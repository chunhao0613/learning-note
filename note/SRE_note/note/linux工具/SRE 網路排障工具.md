## SRE 網路排障工具 (Linux Network Tools)

在 Linux 中，網路排障分為兩個流派：「看狀態（靜態）」與「抓封包（動態）」。

| **工具名稱**                                              | **核心定位**                                              | **SRE 實戰常用參數與含義**                                                                                                                                                                                                                                                                                     | **面試/實戰場景**                                                                  |
| ----------------------------------------------------- | ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| **`tcpdump`**                                         | **網路界的顯微鏡**<br><br>  <br><br>(動態側錄經過網卡的封包)            | `-i eth0`：指定監聽哪一張網卡 (interface)。<br><br>  <br><br>`-n`：不把 IP 解析成網域名稱 (極重要！否則指令會卡住)。<br><br>  <br><br>`-nn`：連 Port 都不解析成服務名 (例如顯示 80 而不是 http)。<br><br>  <br><br>`port 80`：只抓 80 port 的封包。<br><br>  <br><br>`host 1.2.3.4`：只抓跟特定 IP 有關的封包。<br><br>  <br><br>`-w file.pcap`：把抓到的封包存成檔案，事後用 Wireshark 看。 | 「客戶說發了請求，但我 Nginx 的 Log 沒看到。」這時用 tcpdump 檢查封包到底有沒有抵達網卡。                      |
| **`ss`**<br><br>  <br><br>(取代舊版 `netstat`)            | **TCP 連線狀態快照**<br><br>  <br><br>(查看目前主機上的連線與 Port 佔用) | `-t`：只顯示 TCP 連線。<br><br>`-u`：只顯示 UDP 連線。<br><br>`-l`：只顯示正在 Listen (監聽) 的 Port。<br><br>`-n`：不解析名稱 (顯示 IP 與數字 Port)。<br><br>  `-p`：顯示是哪個 Process 佔用了這個 Port。<br><br>  ⭐ SRE 必背起手式：`ss -tlnp`                                                                                                            | 網站啟動失敗，報錯 `Address already in use`。用 `ss -tlnp` 抓出是哪個幽靈 Process 佔用了 80 Port。 |
| **`mtr`**<br><br>  <br><br>(取代 `ping` + `traceroute`) | **網路路徑診斷神器**<br><br>  <br><br>(測試到目標 IP 經過的所有節點狀態)    | `-n`：不解析 DNS。<br><br>  <br><br>`-c 100`：送 100 個封包來測試穩定度。                                                                                                                                                                                                                                              | 伺服器連資料庫很慢。用 mtr 看是機房內部網路塞車，還是 ISP 電信商的節點在掉包 (Packet Loss)。                   |
| **`curl`**                                            | **L7 應用層測試刀**<br><br>  <br><br>(模擬 HTTP 客戶端發送請求)      | `-I`：只拿 HTTP Header (測試網站有沒有活著，不下載內容)。<br><br>  <br><br>`-v`：(Verbose) 顯示完整的交握過程與詳細資訊。                                                                                                                                                                                                                | 測試 API 端點是否正常回應 200 OK，或者檢查 SSL 憑證資訊。                                        |
| **`lsof`**                                            | **萬物皆檔案的透視鏡**<br><br>  <br><br>(列出被打開的檔案與網路 Socket)   | `-i :80`：列出所有使用 80 Port 的行程。<br><br>  <br><br>`-iTCP -sTCP:ESTABLISHED`：列出所有已建立的 TCP 連線。                                                                                                                                                                                                              | 當系統報錯「Too many open files」時，用來抓出哪個程式忘記關閉網路連線或檔案。                             |
